# Implementación del plan de proyecto (issues)

Este directorio contiene la implementación operativa del plan para cargar y mantener el backlog en GitHub Issues.

## Contenido
- `labels.md`: catálogo de labels recomendados (tipo, prioridad, estado, dominio)
- `milestones.md`: milestones por fase
- `issue-templates.md`: formato estándar para Epic/Feature/Task
- `backlog-issues.md`: backlog completo agrupado por Epic → Feature → Task

## Uso recomendado
1. Crear labels de `labels.md`
2. Crear milestones de `milestones.md`
3. Crear primero los 13 issues Epic
4. Crear luego los 51 issues Feature y enlazarlos a su Epic
5. Crear luego los 181 issues Task y enlazarlos a su Feature
6. Ejecutar carga por lotes (MVP primero, Premium después)
